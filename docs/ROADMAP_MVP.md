# Roadmap MVP (version initiale)

## Lot 0 — Fondations
- Cadre documentaire produit/technique.
- Structure repo et règles de sécurité.
- Standards de modularité.

## Lot 1 — Socle applicatif
- Bootstrap frontend et backend (sans déploiement).
- Authn/Authz de base (RBAC multi-rôles).
- Modèle entreprises/utilisateurs/modules.

## Lot 2 — Modes d’usage
- Mode Essentiel : dashboard guidé clair.
- Mode Expert : analyses et preuves détaillées.
- Mode Terrain : flux de déclaration rapide.
- Mode Admin : administration SaaS.

## Lot 3 — Imports & Reporting
- Pipeline import Excel/PDF avec validation humaine.
- Génération PDF niveau cabinet.

## Lot 4 — Conformité et réglementation
- Packs réglementaires paramétrables.
- Préparation ISO 9001 / 14001 / 45001.


## Exigences transverses ajoutées (prioritaires)
- Le mode Terrain offline-compatible est une exigence d’architecture dès le départ.
- L’offline complet type PWA peut être progressif, mais la compatibilité structurelle doit être prévue dès le Lot 1.
- Le dashboard de pilotage réel doit arriver tôt dans la roadmap, pas en fin de cycle.
- Les KPI doivent être actionnables (drill-down, filtres, actions) et non simplement affichés.
